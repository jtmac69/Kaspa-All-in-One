## üîÑ Hybrid Approach: Starting with Templates

### Template as Starting Point

You can use templates as a starting point and customize later:

1. **Start with Template**: Choose the closest template to your needs
2. **Deploy and Test**: Get familiar with the services and configuration
3. **Identify Gaps**: Determine what needs to be changed
4. **Gradual Customization**: Make incremental changes
5. **Document Changes**: Keep track of customizations for future reference

### Template Migration Path

#### From Template to Custom
1. **Export Configuration**: Use Management Dashboard to export current config
2. **Switch to Custom**: Use wizard's reconfiguration feature
3. **Import Base Config**: Start with exported template configuration
4. **Make Modifications**: Add, remove, or modify services as needed
5. **Test Thoroughly**: Ensure all changes work as expected

#### Between Templates
1. **Backup Current Setup**: Create backup of current configuration and data
2. **Use Reconfiguration**: Access reconfiguration through Management Dashboard
3. **Select New Template**: Choose different template in wizard
4. **Migrate Data**: Wizard handles data migration where possible
5. **Verify Services**: Ensure all services are working correctly

## üöÄ Getting Started

### With Templates
1. **Launch Wizard**: `docker compose --profile wizard up -d`
2. **Complete System Check**: Let wizard analyze your system
3. **Choose "Use a Template"**: Select the template approach
4. **Select Template**: Choose based on your use case and hardware
5. **Configure**: Fill in template-specific configuration
6. **Install**: Let the wizard handle the installation

### With Custom Setup
1. **Launch Wizard**: `docker compose --profile wizard up -d`
2. **Complete System Check**: Let wizard analyze your system
3. **Choose "Build Custom"**: Select the custom approach
4. **Select Services**: Choose individual services from the grid
5. **Configure Each Service**: Set up each service individually
6. **Review Dependencies**: Ensure all dependencies are met
7. **Install**: Deploy your custom configuration

### Switching Approaches
You can always switch between approaches:
- **Template to Custom**: Use reconfiguration feature in Management Dashboard
- **Custom to Template**: Use wizard's reconfiguration with template selection
- **Between Templates**: Use template migration in reconfiguration
